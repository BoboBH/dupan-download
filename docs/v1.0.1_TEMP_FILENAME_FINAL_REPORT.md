## 🔍 问题分析

### 文件名长度限制
**Windows 文件系统限制**:
- 文件名最大长度：255字符
- 完整路径最大长度：260字符（MAX_PATH）

**实际文件名示例**:
```
Bernstein-Asia Quantitative Strategy：Best of Times，Worst of Times：How to invest amidst the Kshaped polarization in Asia-260703.pdf
```
这个文件名就很容易超过长度限制。

### 之前的问题
- 文件名过长导致移动失败
- 特殊字符处理复杂
- 文件名验证和清理繁琐
- 仍然可能有长度限制问题

## ✅ 用户建议的完美实现

### 核心策略
**使用极短的临时文件名 + 保持原始文件名用于上传**

### 实现细节

#### 1. 临时文件名生成
```python
def id_to_alpha(id):
    result = ""
    while id >= 0:
        result = chr(ord('a') + (id % 26)) + result
        id = id // 26 - 1
        if id < 0:
            break
    return result if result else "a"

# 生成序列：a, b, c, ..., z, aa, ab, ac, ...
temp_suffix = id_to_alpha(temp_id)
temp_file_name = f"temp_{temp_suffix}.pdf"
```

**文件名序列**:
- `temp_a.pdf` (第1个文件)
- `temp_b.pdf` (第2个文件)
- `temp_c.pdf` (第3个文件)
- ...
- `temp_z.pdf` (第26个文件)
- `temp_aa.pdf` (第27个文件)
- `temp_ab.pdf` (第28个文件)
- ...

#### 2. 完整的文件处理流程
```python
# 提取原始文件名
original_file_name = os.path.basename(remote_path)

# 生成本地临时文件名
temp_file_name = f"temp_{temp_suffix}.pdf"
local_path = os.path.join(self.temp_dir, temp_file_name)

# 下载到临时文件名
download(remote_path, local_path)  # remote -> temp_a.pdf

# 上传时使用原始文件名
upload(local_path, original_file_name)  # temp_a.pdf -> 原始文件名.pdf

# 数据库记录使用原始文件名
db.insert(file_name=original_file_name)  # 记录原始文件名

# 清理临时文件
delete(local_path)  # 删除 temp_a.pdf
```

#### 3. 上传路径处理
```python
# SFTP 上传时使用原始文件名
remote_upload_path = os.path.join(
    self.sftp_client.remote_path,
    folder_name,
    original_file_name  # 使用原始文件名，不是临时文件名
).replace('\\', '/')
```
