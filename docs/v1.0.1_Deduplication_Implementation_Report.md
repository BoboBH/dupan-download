### 1. 数据库去重查询方法
**新增方法**: `get_file_log_by_name_and_link()`

**位置**: `src/database/repository.py`

**功能**: 根据文件名、分享链接、目录名查询文件传输日志

```python
def get_file_log_by_name_and_link(self, file_name: str, share_link: str, folder_name: str) -> Optional[FileTransferLog]:
    """
    根据文件名和分享链接获取文件传输日志
    
    查询条件:
    - 文件名相同
    - 分享链接相同
    - 目录名相同
    - 按创建时间降序排序
    - 返回最新的一条记录
    """
    sql = """
    SELECT * FROM file_transfer_log
    WHERE file_name = %s AND share_link = %s AND folder_name = %s
    ORDER BY created_at DESC
    LIMIT 1
    """
```

### 2. 文件处理去重逻辑
**位置**: `src/processor/file_processor.py`

**在 `_process_single_file` 方法开始添加检查**:

```python
# ===== 去重检查：检查文件是否已经成功上传 =====
existing_log = self.db_repo.get_file_log_by_name_and_link(
    file_name=original_file_name,
    share_link=share_link,
    folder_name=folder_name
)

if existing_log and existing_log.transfer_status == 'success':
    logger.info(f"File already uploaded successfully: {original_file_name}")
    logger.info(f"Previous upload time: {existing_log.upload_time}")
    logger.info(f"Skipping download and upload")
    return 'skipped'  # 跳过已上传的文件
elif existing_log and existing_log.transfer_status == 'uploading':
    logger.warning(f"File is currently being uploaded: {original_file_name}")
    logger.info(f"Skipping to avoid duplicate operations")
    return 'skipped'  # 跳过正在上传的文件
elif existing_log:
    logger.info(f"File exists in database with status: {existing_log.transfer_status}")
    logger.info(f"Will retry upload for: {original_file_name}")
    # 继续处理，尝试重新上传
else:
    logger.debug(f"No existing log found for: {original_file_name}")

# ===== 继续正常的上传流程 =====
```

### 3. 去重策略

#### 策略1: 成功上传 → 跳过
```python
if existing_log.transfer_status == 'success':
    # 跳过下载和上传
    return 'skipped'
```

#### 策略2: 正在上传 → 跳过
```python
elif existing_log.transfer_status == 'uploading':
    # 避免重复操作
    return 'skipped'
```

#### 策略3: 失败记录 → 重试
```python
else:
    # 继续处理，尝试重新上传
    # 支持失败重试机制
```

#### 策略4: 无记录 → 正常处理
```python
else:
    # 没有记录，正常处理
```

## 🔧 关键特性

### 1. 多条件匹配
**匹配条件**:
- ✅ 文件名相同
- ✅ 分享链接相同
- ✅ 目录名相同

**原因**: 确保是同一个文件、同一个来源、同一个目标

### 2. 状态检查
**检查状态**:
- `success`: 跳过（已成功上传）
- `uploading`: 跳过（正在处理）
- `failed`: 继续处理（支持重试）
- `pending`: 继续处理

### 3. 详细日志
**日志信息**:
- 查询到的记录信息
- 上传时间
- 跳过原因
- 处理状态

### 4. 统计更新
**统计计数**:
- `success_count`: 成功上传的文件数
- `failed_count`: 失败的文件数
- `skipped_count`: 跳过的文件数 ⭐
