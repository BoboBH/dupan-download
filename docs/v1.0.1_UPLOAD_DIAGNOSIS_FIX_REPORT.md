## ✅ 修复方案

### 1. 改进文件名提取逻辑
**新代码**:
```python
# 改进的文件名提取：处理各种路径格式
if '/' in remote_path:
    # 处理 Unix 风格路径和 UNC 路径
    parts = remote_path.split('/')
    # 找到最后一个非空部分作为文件名
    for part in reversed(parts):
        if part and part.endswith('.pdf'):
            original_file_name = part
            break
    else:
        # 如果没有找到，使用最后一个部分
        original_file_name = parts[-1] if parts else "unknown.pdf"
else:
    # Windows 风格路径
    original_file_name = os.path.basename(remote_path)

# 确保文件名不为空
if not original_file_name:
    original_file_name = "unknown.pdf"
```

### 2. 增强 SFTP 上传错误日志
**新代码**:
```python
except Exception as e:
    logger.error(f"File upload failed: {type(e).__name__}: {e}")
    logger.error(f"Local path: {local_path}")
    logger.error(f"Remote path: {remote_path}")
    logger.debug(f"Upload exception details: {repr(e)}")
    return False
```

### 3. 添加详细的上传调试信息
**新代码**:
```python
logger.info(f"Uploading {original_file_name} to SFTP...")
```
