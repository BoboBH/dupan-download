# 🔧 v1.0.1 BaiduPCS-Go 登录问题修复报告

## 🎯 新发现问题

用户报告的新错误：
```
[ERROR] [2026-07-12 08:45:17] Baidu login failed
[INFO] [2026-07-12 08:45:17] Temporary files cleanup completed
```

尽管 `.env` 和 `cookies.txt` 都是正确的，但登录仍然失败。

## 🔍 根本原因分析

### 1. BaiduPCS-Go 登录命令错误
**问题代码**:
```python
# 原来的错误实现
result = self._run_command([
    'login',
    '--cookies', self.cookies_path  # ❌ 错误：传递文件路径
])
```

**问题**: 代码试图使用 `--cookies` 参数传递 cookies 文件路径，但 BaiduPCS-Go 不支持这种格式。

### 2. BaiduPCS-Go 正确的登录命令
通过 `BaiduPCS-Go.exe login --help` 发现正确的格式：

```bash
# 正确的命令格式
BaiduPCS-Go.exe login -cookies="BDUSS=xxxxx; BAIDUID=yyyyyy; STOKEN=zzzzz; ...."
```

**关键点**:
- 使用 `-cookies=` (单横杠，不是双横杠)
- 参数应该是 cookies 字符串内容，不是文件路径
- cookies 字符串应该是标准的 HTTP cookie 格式

## ✅ 修复方案

### 1. 修复登录逻辑
**新代码**:
```python
def login(self) -> bool:
    """使用cookies登录百度账号"""
    try:
        if not Path(self.cookies_path).exists():
            logger.error(f"Cookies file not found: {self.cookies_path}")
            return False

        # 读取 cookies 文件内容
        with open(self.cookies_path, 'r', encoding='utf-8') as f:
            cookies_content = f.read().strip()

        if not cookies_content:
            logger.error("Cookies file is empty")
            return False

        logger.info(f"Attempting to login with cookies from: {self.cookies_path}")

        # 使用正确的命令格式
        result = self._run_command([
            'login',
            f'-cookies={cookies_content}'  # ✅ 正确：传递 cookies 内容
        ])

        if result['returncode'] == 0:
            logger.info("Login successful")
            return True
        else:
            logger.error(f"Login failed: {result['stderr']}")
            logger.error(f"Login stdout: {result['stdout']}")
            return False

    except Exception as e:
        logger.error(f"Login exception: {e}")
        return False
```

### 2. 修复要点
1. **读取文件内容**: 读取 `cookies.txt` 文件的内容
2. **传递内容而非路径**: 将 cookies 字符串传递给 BaiduPCS-Go
3. **正确的参数格式**: 使用 `-cookies=` 而不是 `--cookies`
4. **增强日志**: 添加更详细的登录日志，便于调试
