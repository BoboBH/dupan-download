## 🔧 代码修复

### 修复前的代码
**位置**: `src/downloader/baidu_client.py` 第272行

```python
# BaiduPCS-Go输出格式：
# 序号    大小         日期    时间               文件名
# 0       7.00MB      2026-07-12 09:12:43  Long filename with spaces.pdf
parts = line.split()  # ❌ 会合并所有连续空格
if len(parts) >= 4:
    # 前4个字段是：序号、大小、日期、时间
    # 剩下的部分组合起来就是文件名（可能包含空格）
    file_name = ' '.join(parts[4:])  # ❌ 重新用单空格连接
    file_size_str = parts[1]
```

**问题分析**:
1. `line.split()` 会将所有连续的空白字符（空格、制表符等）合并为单个分隔符
2. `' '.join(parts[4:])` 会重新用单个空格连接
3. **原始文件名中的多个连续空格丢失**

### 修复后的代码

**位置**: `src/downloader/baidu_client.py` 第272-284行

```python
# BaiduPCS-Go输出格式：
# 序号    大小         日期    时间               文件名
# 0       7.00MB      2026-07-12 09:12:43  Long filename with spaces.pdf
# 使用正则表达式精确匹配前4个字段，保留文件名的原始空格
match = re.match(r'^(\d+)\s+([\d.]+\s*[KBMG]+)\s+(\d{4}-\d{2}-\d{2})\s+(\d{2}:\d{2}:\d{2})\s+(.+)$', line)
if match:
    # 提取字段
    parts = list(match.groups())
    # 前4个字段是：序号、大小、日期、时间
    # 第5个字段是文件名（保留原始空格）
    file_name = parts[4]  # ✅ 保留原始空格
    file_size_str = parts[1]
```

**修复原理**:
1. 使用正则表达式精确匹配前4个字段
2. 使用 `(.+)$` 捕获剩余部分作为文件名
3. **保留文件名中的所有原始空格**

### 正则表达式详解

```regex
^(\d+)\s+([\d.]+\s*[KBMG]+)\s+(\d{4}-\d{2}-\d{2})\s+(\d{2}:\d{2}:\d{2})\s+(.+)$
```

**分组说明**:
1. `(\d+)` - 序号 (如: 31)
2. `([\d.]+\s*[KBMG]+)` - 文件大小 (如: 5.10MB)
3. `(\d{4}-\d{2}-\d{2})` - 日期 (如: 2026-07-12)
4. `(\d{2}:\d{2}:\d{2})` - 时间 (如: 11:04:12)
5. `(.+)` - 文件名 (保留所有原始空格) ⭐

**关键改进**:
- `\s+` 匹配字段间的空白字符（可能多个空格）
- `(.+)` 捕获文件名，保留所有原始空格
